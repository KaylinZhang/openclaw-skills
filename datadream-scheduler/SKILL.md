# 同步任务调度

## 前置检查

**强制**：第一次调用skill时，先确认 DataDream MCP 服务已安装。未安装则立即结束。

## MCP Tools 工具映射

| 操作 | MCP 工具 |
|------|----------|
| 手动触发（调度）同步任务 | `trigger_sync_task` |
| 重跑同步任务 | `rerun_sync_task` |
| 查询任务批次列表 | `list_task_runs` |
| 查询任务批次详情 | `get_task_run` |

| 操作 | 支持 |
|------|------|

## 手动触发（调度）同步任务

**触发**："调度同步任务"、"手动触发任务"、"触发任务"

| 参数 | 必填 | 说明 |
|------|------|------|
| taskId / taskName | 是 | 任务 ID 或名称 |

**流程**：
1. 根据 taskId 或 taskName 查询任务信息
2. 确认任务状态允许触发（非停用、非下线状态）
3. 触发前向用户确认（展示任务基本信息）
4. 调用 MCP 工具触发
5. 返回批次 ID 和任务开始状态

---

## 重跑同步任务

**触发**："重跑同步任务"、"补数据"、"重新运行任务"

| 参数 | 必填 | 说明 |
|------|------|------|
| taskId / taskName | 是 | 任务 ID 或名称 |
| runId | 否 | 指定的运行批次 ID，不填则默认重跑最新批次 |
| startDate | 否 | 重跑起始日期（格式 `YYYY-MM-DD`） |
| endDate | 否 | 重跑结束日期（格式 `YYYY-MM-DD`），不填则默认当天 |

**流程**：
1. 根据 taskId 或 taskName 查询任务信息
2. 确认任务状态允许重跑
3. 收集重跑参数（runId / 日期范围）
4. 重跑前向用户确认（展示重跑范围和批次信息）
5. 调用 MCP 工具重跑
6. 返回新批次 ID

---

## 查询任务批次列表

**触发**："查看任务批次"、"查询批次记录"、"运行记录"

| 参数 | 说明 |
|------|------|
| taskId / taskName | 任务 ID 或名称 |
| status | 按批次状态筛选（成功/失败/运行中/等待） |
| dateRange | 时间范围筛选（格式 `YYYY-MM-DD ~ YYYY-MM-DD`） |
| page / pageSize | 分页参数 |

**流程**：
1. 调用 MCP 工具查询任务批次列表
2. 按筛选条件过滤（如有）
3. 展示批次列表：批次ID、触发时间、状态、运行时长、错误信息（如有）

---

## 查询任务批次详情

**触发**："批次详情"、"查看某个批次"、"运行详情"

| 参数 | 必填 | 说明 |
|------|------|------|
| runId | 是 | 批次 ID |

**流程**：
1. 调用 MCP 工具根据 runId 查询批次详情
2. 展示完整信息：批次ID、关联任务、触发时间、运行状态、各阶段耗时、错误日志（如失败）、数据量统计

---

## 约束与注意事项

- **项目归属**：所有操作默认在当前项目下进行
- **状态约束**：已停用或已下线的任务不允许触发或重跑
- **重跑风险**：重跑会覆盖历史数据，执行前必须向用户明确说明影响范围
- **批次保留**：历史批次记录按系统策略保留，超期后自动清理
- **修改与删除要求**：未明确授权可修改的参数均不可修改，未明确授权可删除的参数均不可删除
