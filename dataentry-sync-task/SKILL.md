# DataDream 同步任务 Skill

创建和修改 DataDream 平台上的同步任务，当前支持数据填报2Hive链路类型。

## 前置检查

**强制前置检查**：先检查是否已安装 DataDream 相关的 MCP 服务（提供同步任务管理工具的 MCP 服务）。
- 若未安装，**立即结束流程**，不继续执行任何业务步骤。

## 核心概念

- **同步任务**：将数据从来源端同步到去向端的任务，当前支持「数据填报2Hive」链路类型。
- **数据填报2Hive**：将数易数据填报表中的数据同步到 Hive 表。
- **来源数据源**：数易数据填报类型的数据源，绑定了一个数易项目空间。
- **去向数据源**：Hive 类型数据源，默认使用当前项目库。
- **自动建表**：系统自动创建目标 Hive 表，表结构根据来源字段自动生成。
- **字段映射**：来源字段与去向字段的对应关系。
- **调度配置**：任务的执行周期和触发方式（手动触发或周期调度）。
- **业务基线**：任务关联的业务基线，用于保障数据产出时效。

## 当前版本能力边界

### 支持的能力
- 创建同步任务（数据填报2Hive 类型）
- 修改同步任务（字段映射、告警配置、加密脱敏等）

### 不支持的能力
- 删除、上下线、启停、重跑等管理操作（由 datadream-task-management skill 提供）
- 其他链路类型（MySQL2Hive、Kafka2Hive 等，待后续版本扩展）

## 同步任务模型

```
同步任务
├── 基本信息
│   ├── id: 任务唯一标识（系统生成）
│   ├── name: 任务名称（必填，不允许重名）
│   ├── description: 任务描述（默认自动生成）
│   ├── owner: 任务负责人（默认创建人）
│   ├── linkType: 链路类型（当前仅支持 "数据填报2Hive"）
│   └── projectId: 所属项目ID（默认当前项目）
│
├── 来源配置
│   ├── datasourceId: 来源数据源（必填，数易数据填报类型）
│   ├── dataTableId: 来源数据填报表（必填）
│   ├── isMultiSheet: 是否多sheet（必填）
│   └── sheetName: 指定同步的sheet（多sheet时必填）
│
├── 去向配置
│   ├── datasourceId: 去向Hive数据源（默认当前项目库）
│   ├── autoCreateTable: 是否自动建表（必填）
│   ├── tableName: 目标表名（自动建表时必填；已有表时从列表选择）
│   └── lifecycle: 表生命周期（自动建表时默认30天）
│
├── 字段映射
│   ├── mappings: 映射关系列表
│   │   └── { sourceField, targetField, dataType }
│   └── 默认：全部字段映射，自动建表时去向字段英文名与来源一致
│
├── 调度配置
│   ├── triggerType: 手动触发 / 周期调度（必填）
│   ├── scheduleType: 周期类型（天/周/月/小时，周期调度时必填）
│   ├── startTime: 最早执行时间（默认：天→00:00，周→周一00:00，月→1号00:00）
│   ├── retryCount: 失败重试次数（默认3次）
│   └── baselineId: 关联业务基线（必填）
│
├── 性能配置
│   └── throttleLimit: 限流值（默认10M/s）
│
├── 告警配置
│   ├── errorRecordLimit: 错误记录数超上限告警（默认0条）
│   ├── writeRowLimit: 写入行数超下限告警（默认0条）
│   ├── alertReceivers: 告警接收人（默认任务负责人，可选值班表/项目成员）
│   ├── alertMethods: 告警方式（默认继承任务等级，可选：电话/短信/DChat/邮件）
│   ├── alertGroup: 告警组（可选：odin值班组/DChat群聊，默认无）
│   ├── quietHours: 告警免打扰时段（默认无，即全天通知）
│   └── phoneQuietHours: 电话免打扰时间范围（默认无）
│
├── 加密脱敏配置
│   └── rules: 字段级脱敏规则列表
│       └── { field, type: 加密/hash脱敏, clearOriginal: 是否置空原字段 }
│
└── 状态
    └── status: 草稿/已提交（预检查通过后）
```

## 操作参考

### 1. 创建同步任务

**触发意图**：用户说"创建同步任务"、"新建同步任务"、"建一个数据填报2hive任务"等

**必填参数**：

| 参数 | 说明 |
|------|------|
| sourceDatasource | 来源数据源（数易数据填报类型） |
| sourceDataTable | 来源数据填报表 |
| isMultiSheet | 是否多sheet |
| sheetName | 指定同步的sheet（多sheet时必填） |
| autoCreateTable | 是否自动建表 |
| tableName | 目标表名（自动建表时必填；已有表时从列表选择） |
| fieldMappings | 字段映射（非自动建表时必填指定） |
| triggerType | 手动触发 / 周期调度 |
| scheduleType | 周期类型（周期调度时必填） |
| baselineId | 关联业务基线 |

**默认参数**：

| 参数 | 默认值 |
|------|--------|
| targetDatasource | 当前项目库 |
| lifecycle | 30天（自动建表时） |
| description | 自动生成 |
| owner | 创建人 |
| fieldMappings | 全部字段，去向字段英文名与来源一致 |
| startTime | 天→00:00，周→周一00:00，月→1号00:00 |
| retryCount | 3次 |
| throttleLimit | 10M/s |
| errorRecordLimit | 0条 |
| writeRowLimit | 0条 |
| alertReceivers | 任务负责人 |
| alertMethods | 继承任务等级 |
| alertGroup | 无 |
| encryptRules | 无 |
| quietHours | 无 |
| phoneQuietHours | 无 |

**创建流程**：

1. **收集来源信息**
   - 选择来源数据源（数易数据填报类型）
   - 选择来源数据填报表
   - 确认是否多sheet，多sheet时需指定同步的sheet
   - 校验：暂不支持多个sheet同时写入一张表

2. **收集去向信息**
   - 选择去向Hive数据源（默认当前项目库）
   - 确认是否自动建表
     - 自动建表：输入目标表名，设置生命周期（默认30天）
     - 已有表：从当前Hive数据源下选择目标表

3. **配置字段映射**
   - 自动建表：默认全部字段映射，去向字段英文名与来源一致，用户可按需增删
   - 已有表：需手动指定字段映射关系

4. **配置调度**
   - 选择触发方式：手动触发 / 周期调度
   - 周期调度时：选择周期类型（天/周/月/小时）
   - 设置最早执行时间（按周期类型有默认值）
   - 关联业务基线

5. **配置告警**（使用默认值，用户可修改）
   - 错误记录数告警阈值
   - 写入行数告警阈值
   - 告警接收人、告警方式
   - 告警组、免打扰时段等

6. **配置加密脱敏**（可选，默认无）

7. **预��查**
   - 系统发起预检查，检查常见错误原因
   - 检查项包括：来源表是否开启填报、来源表是否有数据、字段映射是否合法等

8. **确认创建**
   - 向用户展示完整配置信息
   - 用户确认后提交创建
   - 返回创建结果（任务ID、状态）

**参考文档**：创建过程中如有疑问，查阅 [数据填报2Hive使用手册](https://cooper.didichuxing.com/knowledge/share/book/YcSAIftpNcle/2204653102999)

### 2. 修改同步任务

**触发意图**：用户说"修改同步任务"、"编辑同步任务"、"改一下字段映射"等

**可修改字段**：
- 任务名称、描述
- 字段映射（可增加映射，已存在的映射不支持修改）
- 加密脱敏配置
- 告警配置
- 调度配置（调度code、周期不支持修改）

**不可修改字段**：
- 来源去向信息
- 任务类型、链路类型

**流程**：

1. 查询任务当前信息
2. 收集用户要修改的内容
3. 校验修改项是否允许
4. 向用户确认修改内容
5. 调用 MCP 工具更新
6. 返回修改结果

## 约束与注意事项

- **项目归属**：所有同步任务操作默认在当前项目下进行，无需用户额外指定项目
- **来源去向不可变**：同步任务的来源数据源、来源表、去向表一旦创建不可修改
- **多sheet限制**：暂不支持多个sheet同时写入一张Hive表
- **空表限制**：来源数据填报表必须有有效数据，空表不支持同步
- **填报开启**：来源数据填报表必须在数易侧开启填报，否则同步会报错
- **字段映射变更**：数据填报表增加字段后，需手动修改同步任务的字段映射，Hive表才会自动添加字段
- **预检查**：创建时系统会发起预检查，检查常见错误原因提前规避
- **参考文档**：[数据填报2Hive使用手册](https://cooper.didichuxing.com/knowledge/share/book/YcSAIftpNcle/2204653102999)
