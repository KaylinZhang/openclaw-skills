# 数据源配置规范

## 一、MySQL 数据源

### 配置字段

| 字段 | 说明 | 必填 | 示例 |
|------|------|------|------|
| 数据源名称 | 唯一标识名称 | ✅ | ds_mysql_prod |
| 维护方式 | DBA维护 / 自有 | ✅ | DBA维护 |
| 账号用途 | 读 / 写 | ✅ | 读 |
| RDS服务名 | RDS 实例标识 | ✅ | rm-xxx.mysql.rds.aliyuncs.com |
| 数据库名 | database 名称 | ✅ | order_db |
| 账号名 | 格式：账号名(机房ip:port) | ✅ | readonly(10.0.0.1:3306) |
| 连接密码 | 仅写账号需要 | ⚠️ | ******** |

### 读取 vs 写入账号

| 类型 | 说明 | 权限要求 |
|------|------|---------|
| 读账号 | 用于同步数据到 Hive | SELECT 权限 |
| 写账号 | 用于回写数据到 MySQL | INSERT/UPDATE/DELETE 权限 |

### AI 对话示例

```
AI：你的 MySQL 数据源是哪个？
    - 名称：ds_mysql_prod
    - 账号用途：读
    - RDS服务名：rm-xxx.mysql.rds.aliyuncs.com
    - 数据库名：order_db
    - 账号名：readonly(10.0.0.1:3306)
    密码是多少？（用于连接测试）
```

## 二、Hive 数据源

### 配置说明

Hive 数据源**无需单独配置**，采用以下策略：

| 策略 | 说明 |
|------|------|
| 默认项目库 | 默认使用当前项目所属的 Hive 库 |
| 项目隔离 | 每个项目对应一个 Hive 数据源 |
| 无需注册 | 系统自动关联 |

### 特殊情况

如果需要访问其他项目的 Hive 数据：
- 需要单独注册跨项目数据源
- 需要目标项目的授权

## 四、MQ 数据源（ddMQ）

### 配置说明

ddMQ 采用**默认配置**，**无需单独注册**：

| 策略 | 说明 |
|------|------|
| 默认数据源 | 系统内置一个 ddMQ 数据源 |
| 无需配置 | 直接使用，无需注册 |
| 访问控制 | 通过项目权限控制 |

### 特殊说明

- ddMQ 是公司内部消息队列
- 消息格式和 Topic 需要用户确认
- 需要处理消息消费位点

## 八、数据源使用限制

### Agent 模式下的数据源管理

在 Agent 模式下，数据源不需要提前配置名称，而是通过对话动态发现：

```
用户：帮我把订单表同步到 Hive

AI：好的，我来确认数据源。
    - 你的 MySQL 数据源是哪个？
    - 密码是多少？
    - Hive 用当前项目吗？
```

### 敏感信息处理

| 信息类型 | 处理方式 |
|----------|----------|
| 密码 | 不存储、不展示、需要时输入 |
| 连接信息 | 可以存储，需要时可查询 |
| AccessKey | 安全存储，需要授权 |

## 九、数据源发现流程

```
┌─────────────────────────────────────────────────────────────┐
│                    数据源发现流程                              │
└─────────────────────────────────────────────────────────────┘

Step 1: 调用 datasource/list.py 获取可用数据源
        ↓
Step 2: 如果数据源存在 → 选择使用
        如果数据源不存在 → 提示注册或询问其他方案
        ↓
Step 3: 确认数据源连接信息
        - 密码（仅首次或未存储时）
        - 其他连接参数
        ↓
Step 4: 测试连接
        - 连接成功 → 继续
        - 连接失败 → 提示错误，重新确认
```
