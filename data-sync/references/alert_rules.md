# 告警规则说明

## 一、告警类型

| 告警类型 | 说明 | 严重程度 | 默认行为 |
|----------|------|----------|----------|
| 任务失败 | 同步任务执行失败 | 高 | 立即通知 |
| 数据延迟 | 数据延迟超过阈值 | 中 | 延迟 2 小时后通知 |
| 数据量异常 | 数据量与预期差异大 | 中 | 次日通知 |
| 数据质量 | 空值率、重复率异常 | 中 | 次日通知 |

---

## 二、通知渠道

| 渠道 | 说明 | 适用场景 |
|------|------|----------|
| DC | DataCenter 内部消息 | 常规通知 |
| 电话 | 自动拨打电话 | 紧急告警、任务失败 |
| 短信 | 短信通知 | 任务失败（无网络时） |
| 邮件 | 邮件通知 | 数据质量报告 |

### 渠道优先级

| 严重程度 | 渠道组合 |
|----------|----------|
| 紧急（任务失败） | DC + 电话 |
| 高（数据延迟 > 2h） | DC |
| 中（数据量异常） | DC |
| 低（数据质量） | 邮件 |

---

## 三、告警规则配置

### 默认规则

```yaml
# 默认告警配置
alert:
  task_failure:
    enabled: true
    channels: [DC, phone]
    receivers: [任务负责人]
    
  data_delay:
    enabled: true
    threshold_minutes: 120  # 2 小时
    channels: [DC]
    receivers: [任务负责人]
    
  data_quality:
    enabled: false  # 默认关闭，需要时开启
    rules:
      null_ratio_threshold: 0.1      # 空值率不超过 10%
      duplicate_ratio_threshold: 0.01  # 重复率不超过 1%
      count_diff_threshold: 0.2      # 数据量差异不超过 20%
    channels: [DC]
    receivers: [任务负责人]
```

### 自定义规则

用户可以自定义告警规则：

```
用户：帮我配置告警，要求：
      - 任务失败时除了通知我，还要通知组长
      - 数据延迟超过 1 小时就通知
      - 晚上 10 点到早上 8 点不要打电话

AI：好的，生成以下告警配置：

    【告警规则】
    - 任务失败：DC + 电话给 [你, 组长]
    - 数据延迟 > 1h：DC 给 [你]
    - 免打扰时段：22:00 - 08:00（仅电话）
```

## 四、告警免打扰

### 配置格式

```yaml
quiet_hours:
  start: "22:00"
  end: "08:00"
  channels: [phone]  # 仅电话免打扰
```

### 行为说明

| 时段 | DC 通知 | 电话通知 |
|------|---------|----------|
| 白天（08:00-22:00） | ✅ 正常通知 | ✅ 正常拨打 |
| 夜间（22:00-08:00） | ✅ 正常通知 | ❌ 延迟到 08:00 |

### 紧急告警

紧急告警（如任务失败）即使在免打扰时段也会通过电话通知。

## 六、告警模板

### 任务失败告警

```
【同步任务失败】[任务名称]

任务：order_sync
状态：失败
时间：2026-04-12 10:30:00
错误：Java.lang.OutOfMemoryError
负责人：xxx

操作：点击查看详情 / 立即重跑
```

### 数据延迟告警

```
【数据延迟告警】[任务名称]

任务：order_sync
状态：数据延迟
延迟时间：3 小时
最后运行时间：2026-04-12 07:30:00
预期数据时间：2026-04-12 10:00:00

操作：点击查看详情
```

### 数据量异常告警

```
【数据量异常】[任务名称]

任务：order_sync
预期数据量：100,000 条
实际数据量：50,000 条
差异：-50%
时间：2026-04-12

操作：点击查看详情
```

## 七、告警规则优先级

| 优先级 | 告警类型 | 通知方式 | 时效性 |
|--------|----------|----------|--------|
| P0 | 任务失败 | DC + 电话 | 立即 |
| P1 | 数据延迟 > 2h | DC | 延迟通知 |
| P2 | 数据量异常 > 20% | DC | 次日 |
| P3 | 数据质量异常 | 邮件 | 次日 |
