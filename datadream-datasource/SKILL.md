# DataDream 数据源管理 Skill

管理 DataDream 平台上的数据源，支持多种数据源类型的创建、查询、修改、删除操作。

## 前置检查

**强制前置检查**：先检查是否已安装 DataDream 相关的 MCP 服务（提供数据源管理工具的 MCP 服务）。
- 若未安装，**立即结束流程**，不继续执行任何业务步骤。

## 核心概念

- **数据源**：DataDream 平台中注册的外部数据来源，用于为同步任务提供来源或去向数据。数据源归属于当前项目，不指定项目时默认在当前项目下操作。
- **数据源类型**：支持多种数据源类型，每种类型的配置参数不同。当前支持「数易数据填报」，后续会扩展 MySQL、Kafka等。

## 支持的数据源类型

| 类型 | 用途 | 当前支持 |
|------|------|----------|
| 数易数据填报 | 作为同步任务来源，读取数易数据填报表 | ✅ 支持 |
| ... | 其他类型持续扩展中 | - |

## 当前版本能力边界

### 支持的能力
- 创建数据源（数易数据填报类型）
- 查询数据源列表
- 查询数据源详情
- 修改数据源（按类型有不同的可修改字段）
- 删除数据源（需无关联的同步任务）

### 不支持的能力
- 其他数据源类型的创建（MySQL、Kafka 等，待后续版本扩展）
- 数据源连通性测试（待 MCP 工具支持后开放）

## 通用数据源模型

```
数据源（DataSource）
├── id: 数据源唯一标识（系统生成）
├── name: 数据源名称（选填，不填时自动生成）
├── type: 数据源类型（必填）
├── config: 类型特定的配置（按类型不同而不同）
├── projectId: 所属项目ID（默认当前项目）
└── createdAt: 创建时间
```

## 操作参考

### 1. 创建数据源

**触发意图**：用户说"创建数据源"、"新增数据源"、"注册数据源"等

**创建流程**：

```
步骤1: 判断数据源类型
    │
    ├─ 用户明确指定类型 → 使用指定类型
    │
    └─ 用户未指定类型 → 询问用户要创建什么类型的数据源
         │
         └─ 展示支持的数据源类型列表供用户选择

步骤2: 根据类型分支到对应的创建流程
    │
    ├─ 类型 = "数易数据填报" → 执行【创建数易数据填报数据源】流程
    │
    ├─ 类型 = "MySQL" → 执行【创建 MySQL 数据源】流程（待支持）
    │
    └─ ... 其他类型

步骤3: 确认并创建
    │
    └─ 向用户展示完整配置，确认后调用 MCP 工具创建
```

#### 1.1 创建「数易数据填报」数据源

**类型标识**：`datae_dataentry`

**用途**：作为同步任务的来源，读取指定数易项目空间下的数据填报表。

**参数**：

| 参数 | 必填 | 说明 |
|------|------|------|
| name | 否 | 数据源名称。不填时自动生成，格式：`{项目空间名称}_数据填报` |
| shuyiProjectSpace | 是 | 绑定的数易项目空间，用于读取该空间下的数据填报表 |

**流程**：

1. 收集数易项目空间信息：
   - 如果用户已提供，直接使用
   - 如果未提供，查询可用的数易项目空间列表供用户选择
2. 数据源名称处理：
   - 如果用户已提供，直接使用
   - 如果未提供，基于项目空间名称自动生成
3. 向用户确认最终信息（名称、类型、绑定的项目空间）
4. 确认后调用 MCP 工具创建数据源
5. 返回创建结果

**可修改字段**：仅允许修改名称，绑定的项目空间不可修改。

**环境约束**：DataDream 和数易必须同环境。

#### 1.2 创建「MySQL」数据源（待支持）

**类型标识**：`mysql`

**用途**：作为同步任务的来源或去向，连接 MySQL 数据库。

**参数**：

| 参数 | 必填 | 说明 |
|------|------|------|
| name | 否 | 数据源名称。不填时自动生成，格式：`{host}_{database}` |
| host | 是 | 数据库主机地址 |
| port | 是 | 端口号 |
| database | 是 | 数据库名 |
| username | 是 | 用户名 |
| password | 是 | 密码 |

*（此类型待后续版本支持）*

### 2. 查询数据源列表

**触发意图**：用户说"数据源列表"、"查看数据源"、"我有哪些数据源"等

**参数**：

| 参数 | 必填 | 说明 |
|------|------|------|
| type | 否 | 按类型筛选 |
| keyword | 否 | 按名称关键词筛选 |

**流程**：

1. 调用 MCP 工具查询当前项目下的数据源列表
2. 按筛选条件过滤（如有）
3. 展示数据源基本信息：名称、类型、关键配置、创建时间

### 3. 查询数据源详情

**触发意图**：用户说"数据源详情"、"看一下某个数据源"等

**参数**：

| 参数 | 必填 | 说明 |
|------|------|------|
| id/name | 是 | 数据源ID或名称 |

**流程**：

1. 根据ID或名称查询数据源详情
2. 展示完整信息：名称、类型、类型特定配置、创建时间、关联的同步任务数量

### 4. 修改数据源

**触发意图**：用户说"修改数据源"、"编辑数据源"等

**可修改字段（按类型不同）**：

| 数据源类型 | 可修改字段 |
|------------|------------|
| 数易数据填报 | 仅名称 |
| MySQL | 名称、host、port、database、username、password |
| ... | ... |

**流程**：

1. 查询数据源当前信息和类型
2. 根据类型确定可修改字段
3. 收集用户要修改的内容
4. 校验修改项是否允许
5. 向用户确认修改内容
6. 调用 MCP 工具更新
7. 返回修改结果

### 5. 删除数据源

**触发意图**：用户说"删除数据源"、"移除数据源"等

**前置约束**：**数据源下必须没有关联的同步任务**才能删除。如有关联任务，需提示用户先处理关联任务。

**参数**：

| 参数 | 必填 | 说明 |
|------|------|------|
| id/name | 是 | 数据源ID或名称 |

**流程**：

1. 查询数据源信息
2. 检查该数据源是否有关联的同步任务
   - 如有关联任务：提示用户"该数据源下存在 N 个同步任务，请先删除或解绑相关同步任务后再删除数据源"，并列出关联任务
   - 如无关联任务：继续下一步
3. 向用户确认删除操作
4. 确认后调用 MCP 工具删除
5. 返回删除结果

## 约束与注意事项

- **项目归属**：所有数据源操作默认在当前项目下进行，无需用户额外指定项目
- **类型不可变**：数据源类型一旦创建不可修改
- **删除保护**：有关联同步任务的数据源不可删除，防止破坏已有同步链路
- **名称唯一**：同一项目下数据源名称不允许重复
- **类型特定约束**：每种数据源类型有各自的约束条件（如数易数据填报的环境一致性要求）
