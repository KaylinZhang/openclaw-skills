# 数据源管理

## 前置检查

**强制**：先确认 DataDream MCP 服务已安装。未安装则立即结束。

## MCP Tools 工具映射

| 操作 | MCP 工具 |
|------|---------|
| 创建数据源 | `create_datasource` |
| 查询数据源列表 | `list_datasources` |
| 查询数据源详情 | `get_datasource` |
| 修改数据源 | `update_datasource` |
| 删除数据源 | `delete_datasource` |

---

## 能力边界

| 操作 | 支持状态 |
|------|---------|
| 创建「数易数据填报」数据源 | ✅ |
| 创建「Cooper」数据源 | ✅ |
| 查询数据源列表/详情 | ✅ |
| 修改数据源 | ✅（按类型限定可改字段） |
| 删除数据源 | ✅（需无关联任务） |
| MySQL/Kafka等其它类型 | ❌ 待扩展 |

## 数据源模型

```
DataSource
├── id          # 系统生成
├── name        # 名称（不填自动生成）
├── type        # 数易数据填报(datae_dataentry) / Cooper(cooper)
├── config      # 类型特定配置（见各类型说明）
└── projectId   # 所属项目（默认当前项目）
```

## 创建数据源

**触发**："创建数据源"、"新增数据源"、"注册数据源"

### 类型「数易数据填报」（类型标识：`datae_dataentry`）

**用途**：作为同步任务来源，读取数易项目空间下的数据填报表。

| 参数 | 必填 | 说明 |
|------|------|------|
| name | 否 | 不填时自动生成：`{项目空间名称}_数据填报` |
| shuyiProjectSpace | 是 | 绑定的数易项目空间 |

**流程**：
1. 收集项目空间信息（未提供则查询列表供选择）
2. 确认名称和项目空间
3. 调用 MCP 创建
4. 返回结果

**约束**：
- DataDream 和数易必须同环境
- 名称唯一（同一项目下）
- 类型不可变
- 仅可修改名称

---

### 类型「Cooper」（类型标识：`cooper`）

**用途**：作为同步任务来源，读取 Cooper 团队空间下的协作表格。

| 参数 | 必填 | 说明 |
|------|------|------|
| name | 否 | 不填时自动生成：`Cooper_{空间ID}` |
| spaceId | 是 | Cooper 团队空间 ID，从 URL 的 `team-file` 后数字串获取 |
| token | 是 | Cooper OpenAPI Token，**禁止猜测**，未提供则引导用户申请 |

> ⚠️ **token 获取方式**：
> 引导用户通过 BPM 申请：https://bpm.didichuxing.com/process/form/bykey/cooper_open_api?tenantId=Cooper
> 申请成功后会收到邮件，包含 Token 信息

**流程**：
1. 收集空间 ID（未提供则引导用户从 URL 获取）
2. **收集 token**（⚠️ 禁止猜测，未提供则引导申请）
3. 确认名称、空间 ID、Token
5. 返回结果

**约束**：
- 可修改字段：名称、Token（空间 ID 不可修改）
- 参考文档 → [references/product-manuals.md](references/product-manuals.md)

## 查询数据源

**触发**："数据源列表"、"查看数据源"

| 参数 | 说明 |
|------|------|
| type | 按类型筛选 |
| keyword | 按名称关键词筛选 |

**流程**：
1. 调用 MCP 工具查询当前项目下的数据源列表
2. 按筛选条件过滤（如有）
3. 展示数据源基本信息：名称、类型、关键配置、创建时间

---

## 查询数据源详情

**触发**："数据源详情"、"看一下某个数据源"

| 参数 | 必填 | 说明 |
|------|------|------|
| id/name | 是 | 数据源 ID 或名称 |

**流程**：
1. 根据 ID 或名称查询数据源详情
2. 展示完整信息：名称、类型、类型特定配置、创建时间、关联的同步任务数量

---

## 修改数据源

**触发**："修改数据源"、"编辑数据源"

按类型限定可改字段：

| 数据源类型 | 可修改字段 |
|------------|------------|
| 数易数据填报 | 仅名称 |
| Cooper | 名称、Token |
| *版本待扩展* MySQL | 名称、host、port、database、username、password | 

**流程**：
1. 查询数据源当前信息和类型
2. 根据类型确定可修改字段
3. 收集用户要修改的内容
4. 校验修改项是否允许
5. 向用户确认修改内容
6. 调用 MCP 工具更新
7. 返回修改结果

---

## 删除数据源

**触发**："删除数据源"、"移除数据源"

**前置约束**：数据源下必须没有关联的同步任务才能删除。

**流程**：
1. 查询数据源信息
2. 检查该数据源是否有关联的同步任务
   - 如有关联任务：提示用户"该数据源下存在 N 个同步任务，请先删除或解绑相关同步任务后再删除数据源"，并列出关联任务
   - 如无关联任务：继续下一步
3. 向用户确认删除操作
4. 确认后调用 MCP 工具删除
5. 返回删除结果

---

## 约束与注意事项

- **项目归属**：所有数据源操作默认在当前项目下进行，无需用户额外指定项目
- **删除保护**：有关联同步任务的数据源不可删除，防止破坏已有同步链路
- **名称唯一**：同一项目下数据源名称不允许重复
